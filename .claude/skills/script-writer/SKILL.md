# Script Writer Skill

客戶給你需求問卷後，依照以下流程產出完整的提案腳本。
**此 Skill 整合 4 個 Agent 知識庫的專業判斷，每個階段應對應各 Agent 的知識基礎。**

---

## 觸發條件

使用者提供以下任一輸入時啟動此 Skill：
- 客戶需求問卷的 `.docx` 檔案路徑
- 直接貼上問卷的 Q1–Q5 內容

---

## 多 Agent 知識庫整合架構

產出腳本時，依照以下 Agent 分工各司其職：

| Agent | 知識庫路徑 | 負責腳本的哪個部分 |
|-------|-----------|-----------------|
| **市場分析師** | `Agent/市場分析師/knowledge/廣告專案市場分析知識庫.md` | 解讀問卷、分析競爭優勢排序、定義目標觀眾 |
| **創意總監** | `Agent/創意總監/knowledge/影片廣告創意製作知識庫.md` | 定義核心訊息框架、撰寫 VO 旁白、決定情感走向 |
| **導演** | `Agent/導演/knowledge/廣告影片製作知識庫.md` | 規劃影像說明（@指示）、時間碼配置、視覺執行策略 |
| **視覺設計師** | `Agent/視覺設計師/knowledge/動態設計與影音廣告視覺知識庫.md` | 決定字卡使用策略、視覺風格定義、色調建議 |

---

## Step 1：讀取問卷（市場分析師視角）

若使用者提供 `.docx` 路徑，用 python-docx 解析：

```python
from docx import Document
doc = Document("問卷路徑.docx")
for table in doc.tables:
    for row in table.rows:
        print(row.cells[0].text, "→", row.cells[1].text)
```

萃取以下 5 題答案，並進行市場分析師層次的解讀：

| 題號 | 欄位 | 市場分析師解讀任務 |
|------|------|------------------|
| Q1 | 核心訴求 | 識別品牌定位核心，決定影片的敘事框架主題 |
| Q2 | 競爭優勢 | 分析 2–3 個優勢的市場說服力，排序影片露出優先次序 |
| Q3 | 關鍵競爭力 | 找出「沒有競品能複製的護城河」，這是最核心的信任建立點 |
| Q4 | 必要場景 | 解讀為客戶希望傳遞的市場信號（規模感或技術可信度的視覺佐證） |
| Q5 | 目標市場 | 區分現有客群（認同感）vs 未來開拓方向（主要呼籲對象） |

## Step 2：確認客戶名稱

從檔名（`客戶需求問卷_XXX.docx`）或使用者說明取得客戶名稱。

---

## Step 3：建立核心訊息（創意總監視角）

在產出分鏡前，先建立整支影片的敘事框架：

**核心訊息須包含三個層次：**

| 層次 | 說明 |
|------|------|
| **主題框架** | 以一個具象概念包裝品牌精神（例：「The Crucible 實驗箱」）|
| **一句標語** | 精煉的品牌主張（例：《極致恆溫的最高標準》）|
| **說明段落** | 50 字內說明影片核心概念與主視覺方向 |

**原則：** 核心訊息不是功能清單，而是讓品牌「有理由存在」的故事框架。

## Step 4：產出腳本（整合 4 Agent 知識庫）

### 輸出格式（完整結構）

```
# 【客戶名稱】提案腳本

## 企業形象影片

**核心訊息：**
（主題框架 + 一句標語 + 50字內說明）

**故事大綱：**
（3–5句，說明影片整體敘事邏輯與情感走向）

**風格 / 情緒：**

**目標觀眾：**
（從 Q5 衍生：現有客群（守住）→ 未來開拓方向（主要呼籲對象））

## 影片規格

- 片長：2分半影片一部  
- 規格：比例 16:9，影片有效畫素至少 2160p（4K），FULL HD 輸出格式為 MP4  
- 語版：中文配音、中文字卡、中文字幕  
- 交付：有字幕版本、無字幕字卡版本  
- 演員：（依客戶需求填寫，預設為台灣演員 1 位）

*圖片作為參考構圖
```

## 7 段架構邏輯（含各 Agent 知識來源）

每段對應問卷欄位如下，可依品牌性質微調段落順序：

| 段落 | 對應問卷 | 功能 | 建議時間碼 | Agent 主責 |
|------|----------|------|-----------|-----------|
| 1. 核心開場 | Q1 | 情感鉤子，建立品牌第一印象 | 0:00–0:20 | 創意總監（敘事框架）+ 導演（英雄鏡頭） |
| 2. 競爭力 A | Q2（優勢 1+2） | 第一組差異化陳述 | 0:20–0:40 | 市場分析師（優勢排序）+ 視覺設計師（字卡策略） |
| 3. 競爭力 B | Q2（優勢 3）+ Q3 | 關鍵競爭力強化，最核心的賣點 | 0:40–1:05 | 市場分析師（護城河識別）+ 導演（實測視覺） |
| 4. 領先工藝 | Q4 | 視覺落地，展示產品/服務的具體畫面 | 1:05–1:25 | 導演（拍攝指示）+ 視覺設計師（技術規格字卡） |
| 5. 整合實力 | Q2 + Q3 延伸 | 全方面服務能力，建立信任感 | 1:25–1:40 | 導演（人員情境鏡頭）|
| 6. 願景總結 | Q1 + Q5 | 升華品牌高度，連結市場願景 | 1:40–1:55 | 市場分析師（現有/未來市場）+ 創意總監（情感升華） |
| 7. 品牌精神 | 客戶名稱 + 標語 | Slogan + LOGO，情感收尾 | 1:55–2:00 | 創意總監（Slogan）+ 視覺設計師（LOGO動畫規格） |

## VO 旁白語氣指引（創意總監知識庫）

- **語感**：沉穩、有力、略帶詩意，不過度煽情
- **句型**：多用短句、並列句，避免長句堆疊
- **節奏**：每段旁白控制在 40–60 字，讀出來約 15–20 秒
- **開場句**：通常是一個強烈的品牌宣言或場景描述
- **結尾句**：每段收尾留有餘韻，避免直白的廣告感

**範例風格對照：**
- ✅ 「一片庫板，凝聚了 20 年的研發與遠見。」（開場有力，帶入感強）
- ✅ 「我們阻絕熱能的流動，也守護空間內對溫控的精準。」（技術感 + 詩意）
- ✅ 「冷鏈是食品安全、醫療品質與半導體製程的根基。」（市場願景升華）
- ❌ 「我們的產品品質非常好，請大家多多支持。」（過於直白）

## 影像說明格式（導演知識庫）

每段的「影像說明」欄用 `@` 開頭列出鏡頭指示，每段 3–5 個：

```
@產品特寫（慢動作）
@工作人員操作畫面
@客戶使用情境
@數據字卡動畫
```

**各段視覺策略參考：**

| 段落 | 建議視覺策略 |
|------|------------|
| 核心開場 | 產品英雄鏡頭（特寫→拉遠，慢動作）+ 數據字卡 |
| 競爭力 A | 全球視野素材 + 合作方品牌素材 + 研發場景 |
| 競爭力 B | 實測/壓力測試畫面（戲劇性構圖）+ 數據視覺化 |
| 領先工藝 | 工廠空拍大景 → 機台特寫 → 製程細節 |
| 整合實力 | 多角度人物情境（品管/設計/客服）|
| 願景總結 | 3個應用場景並列（各有色調差異）+ 員工精神特寫 |
| 品牌精神 | 產品回到特寫 + LOGO動畫（慢動作收尾）|

---

## 字卡使用策略（視覺設計師知識庫）

字卡不是每段都需要，選擇性使用：

| 字卡類型 | 使用段落 | 設計原則 |
|----------|----------|---------|
| 歷史/數字型 | 開場 | 大字號、Sans-Serif 粗體 |
| 品牌聲明型 | 競爭力 A | 精簡一句，突出品牌優勢 |
| 技術規格型 | 領先工藝 | 規格數字突出，去除形容詞 |
| 認證標準型 | 願景總結 | 配合應用場景同步出現 |
| Slogan + LOGO | 品牌精神 | 全畫面靜止，LOGO動畫收尾 |

**純功能段落（品管流程、服務能力）不放字卡，讓畫面與VO說話。**

## 完成後輸出

1. 腳本以 Markdown 格式輸出至對話中供 Masen 確認。
2. 存檔至 `/Users/masen/documents/ap/腳本提案/【客戶名稱】提案腳本.docx`（不產出 .md 檔案）
3. docx 格式規範：
   - 標題 14pt bold #0176BB、段落 10pt black、時間碼 8pt #AEAAAA
   - 表格標題列 #0176BB 底色、白色文字；無外框、內分隔線 #AEAAAA
   - 欄寬（dxa）：段落=1119、字卡=1890、參考圖片=2834、影像說明=2409、旁白=2197
4. 輸出後說明兩份檔案的存檔路徑，並請 Masen 確認內容。
