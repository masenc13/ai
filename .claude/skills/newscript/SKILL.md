# newscript Skill

客戶在會議中確認採用哪一個提案策略後，把那一案發展成**一份含完整分鏡的提案腳本**。
派**導演 + 創意總監**兩位 Sub-Agent 共同創作，依「提案腳本架構（範本）」產出 `.docx`，存回客戶資料夾。

全程**全自動**，三個 Phase 走完後統一報告一次。

---

## 輸入

使用者提供：**一個客戶資料夾路徑**（即 business-proposal 用過的同一個資料夾）。約定結構：

| 項目 | 內容 | 用途 |
|------|------|------|
| `企劃整理/` | 與客戶開會的**結果文字** ＋ **確認採用哪一個提案策略**（編號/名稱） | 決定要發展哪一案、納入客戶在會議裡的具體要求 |
| `【客戶名稱】企業形象提案企劃.pptx` | business-proposal 產出的三策略簡報 | 取出**被選定那一案**的完整內容（Slogan、核心訊息、影片架構、影像方向） |
| `專案內容資料夾/`（影片需求單／問卷） | 客戶原始需求 | **影片規格**（片長/語版/交付/演員）＋**必出現場景/物件** |
| `提案腳本架構（範本）.*` | 腳本格式範本 | 確認輸出版型（已內建於產生器，照著走即可） |

從客戶資料夾名稱取得客戶名稱。

## Phase 1：盤點與閱讀

### Step 1.1：定位

```bash
ls -R "<客戶資料夾路徑>"
```

### Step 1.2：讀「企劃整理」，鎖定選定策略
讀 `企劃整理/` 裡的所有檔案，抓出兩件事：
1. **客戶確認採用哪一個提案策略**（策略一/二/三 或其名稱）。若資料夾裡看不出明確選了哪一案 → **停下來問使用者**（這是少數會停的情況，選錯整份腳本就白做）。
2. **會議中客戶的具體要求/修改意見**（必須反映進腳本，例如指定畫面、調性、不要什麼）。

### Step 1.3：取出選定策略的完整內容
用 python-pptx 讀 `【客戶名稱】企業形象提案企劃.pptx`，擷取**被選定那一案**的：Slogan、核心訊息陳述、影片架構三概念、影像方向（treatment）、市場依據。這是腳本的骨架。

### Step 1.4：讀影片需求單，取規格與必出現場景
讀 `專案內容資料夾/` 的影片需求單：
- **影片規格**：片長、語版（旁白/字卡/字幕語言）、交付、演員 → 一律以**需求單為準**，不要用範本的範例值。
- **必出現場景/物件**：條列出來，**分鏡表必須全部涵蓋**（漏拍是最常見的客訴）。
- 結尾要求（如 LOGO + MADE IN TAIWAN）。

把以上整理成一份**腳本簡報（brief）**，傳給兩位 Sub-Agent。

## Phase 2：導演 + 創意總監共同創作（平行 Sub-Agent）

**同一回合內平行啟動兩個 Sub-Agent**（`general-purpose`）。兩人**都看全部資料、各自產出一份完整腳本草稿**（不硬分工），主線再整合兩稿之長。
進入角色：先讀精簡人設（**絕對路徑**）；該檔還是空模板時才退回讀完整知識庫。

### Sub-Agent A — 創意總監（張雅婷）
- 精簡人設：`/Users/masen/Documents/ap/.claude/skills/business-proposal/references/persona_創意總監.md`
- 退回讀：`/Users/masen/Documents/ap/Agent/創意總監/soul.md` ＋ `…/knowledge/影片廣告創意製作知識庫.md`、`創意總監核心能力知識庫.md`
- 偏重：核心訊息、故事大綱、風格/情緒、目標觀眾、**旁白（VO）與字卡**的文字

### Sub-Agent B — 導演（Sebastian）
- 精簡人設：`/Users/masen/Documents/ap/.claude/skills/business-proposal/references/persona_導演.md`
- 退回讀：`/Users/masen/Documents/ap/Agent/導演/soul.md` ＋ `…/knowledge/廣告影片製作知識庫.md`、`導演核心能力.md`
- 偏重：**影像風格（拍攝手法＋特效風格）**、分鏡的**影像說明（@鏡頭指示）**、節奏與時間碼
- **另外提 1–2 個參考片方向**：分別對應「拍攝手法」與「特效風格」，用**風格描述或具名作品**（例：某品牌形象片、某類紀實片的調性），**不要編造 YouTube URL**；客戶已提供的參考片（如需求單列的範例）也一併納入。這會填進範本的「參考影片」欄。

兩個 Sub-Agent 都要：依選定策略的調性、**涵蓋需求單所有必出現場景**、依需求單語版決定旁白/字卡語言、產出完整分鏡（含段落、時間碼、字卡、影像說明、旁白）。完成後收回主線。

## Phase 3：整合並產出 .docx

主線把兩稿整合成**一份**最終腳本：創意總監的訊息/旁白/字卡為文字主幹，導演的影像風格/鏡頭為影像主幹，彼此對齊到同一條敘事線。

**檢查（整合時務必過一遍）：**
- 需求單的**必出現場景是否全部進到分鏡**？
- 旁白/字卡語言是否符合需求單語版？（如品香＝全英文）
- 結尾是否收 LOGO／指定字卡？
- 分鏡段落是否符合片長（1–2 分鐘約 6–9 段；2–3 分鐘約 8–12 段）？

用內建產生器輸出（**勿手刻 docx**）：
```python
import sys
sys.path.insert(0, "/Users/masen/Documents/ap/.claude/skills/newscript/scripts")
from build_script_doc import build_script_doc

build_script_doc(
    client_name="【客戶名稱】",
    info={  # 見 build_script_doc.py 開頭 schema
        "core": "...", "story": "...", "mood": "...", "audience": "...",
        "spec": {"片長": "...", "規格": "...", "語版": "...", "交付": "...", "演員": "..."},
        "shoot": "拍攝手法段落", "shoot_ref": "參考影片(選填)",
        "fx": "特效風格段落", "fx_ref": "參考影片(選填)",
        "board_title": "【客戶名稱】或片名",
    },
    scenes=[  # 每段一個 dict
        {"seg": "1. 核心開場 0:00–0:15", "card": "字卡", "img": "",
         "visual": "@鏡頭1\n@鏡頭2", "vo": "旁白（語言依需求單）"},
        # ...
    ],
    out_path="<客戶資料夾路徑>/【客戶名稱】提案腳本.docx",
)
```

`info["spec"]` 一律填**需求單的實際規格**。`shoot_ref` / `fx_ref` 填導演提的**參考片方向**（風格描述或具名作品＋客戶提供的參考片；無則留空字串，產生器會自動略過該行）。`scenes` 的 `img` 通常留空（給後續貼參考圖），需要時可填示意文字。
產生器已內建：A4、邊距 0.5"、標題 14pt `#0176BB`、表格標題列 `#0176BB` 白字、無外框內分隔線、5 欄寬（dxa）總和 10449。

## 完成後統一報告

```
✅ 提案腳本完成：【客戶名稱】（採用：提案策略X · 名稱）

📄 腳本：<客戶資料夾路徑>/【客戶名稱】提案腳本.docx
🎬 分鏡：共 N 段／片長 X／語版 Y
✅ 必出現場景：已全部納入（或 ⚠️ 缺 XXX，原因…）
```

需要中英對照時，可接著用 `translate-formal` skill 翻譯旁白與字卡。

## 錯誤處理（不中斷流程）

| 問題 | 處理 |
|------|------|
| 客戶資料夾找不到 | 停下，請使用者確認路徑 |
| `企劃整理/` 看不出選了哪一案 | **停下來問使用者要發展哪一案**（選錯成本最高） |
| 找不到提案簡報 .pptx | 改用 `企劃整理/` 與需求單裡的策略描述，報告標注 |
| 找不到影片需求單 | 規格改用範本預設並標注「⚠️ 規格待客戶確認」 |
| Sub-Agent 某個失敗 | 主線用該角色知識庫自行補上，報告標注 |
| python-docx 未安裝 | `pip install python-docx` 後繼續 |
| 存檔目錄不存在 | 自動 `mkdir -p` |
